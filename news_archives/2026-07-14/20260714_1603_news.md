### 8. 微软优化 Win11 的 Linux 子系统：32MiB 内存余量保护 WSL 主机进程
**发布时间:** Tue, 14 Jul 2026 07:45:11 GMT
**链接:** [https://www.ithome.com/0/976/537.htm](https://www.ithome.com/0/976/537.htm)
**正文:** IT之家 7 月 14 日消息，科技媒体 Neowin 今天（7 月 14 日）发布博文，报道称微软将重构适用于 Windows 11 的 WSL 资源管理架构，降低高负载下发生灾难性故障的可能性。IT之家注：WSL 全称为 Windows Subsystem for Linux，是微软提供的 Windows 子系统，用于在 Windows 中运行 Linux 环境。用户可在其中执行 Linux 命令、编译软件和运行相关工具。在应用该补丁之前，用户在编译软件等高负载任务场景下，会导致虚拟 Linux 环境频繁触发内存不足 (OOM) 事件，不仅让正在运行的应用程序崩溃，还会耗尽 WSL 核心引擎自身运行所需的所有资源。在本次合并补丁中，微软通过沙箱机制，限制虚拟 Linux 环境资源调用，从而确保终有足够的内存和 CPU 资源来维持 WSL 主机进程的运行。在技术实现方面，该补丁计划整合名为 wsl-user 的受限控制组，其内存大小比主机总内存少 32MiB，CPU 核心数比主机总 CPU 容量少 0.01 个核心。这点微小的差距应该足以保证 WSL 能够维持关键的后台通信和系统编排。该媒体还指出 WSL 目前已完全过渡到 Linux cgroup v2，如果旧版工作负载依赖于旧版框架，可能遇到兼容性问题。对于确实需要 Linux cgroup v1 的用户，需要在 .wslconfig 文件中添加 IsolateDistroCgroup = false ，这样可以绕过新版框架的限制。
